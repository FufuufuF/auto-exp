# 规则文件说明

## 简介

本目录包含了 AI 助手完成实验报告任务的所有规则和指南。这些规则文件确保 AI 助手能够：
- 按照规范的流程工作
- 生成高质量的代码和报告
- 保持文件组织的整洁性
- 提供一致的用户体验

## 规则文件列表

### 1. project_overview.mdc
**项目概述规则**

- 项目的整体定位和目标
- 核心功能介绍
- 项目结构说明
- 工作原则

**适用场景**：
- 初次启动项目时阅读
- 理解项目整体架构
- 明确工作职责和定位

### 2. workflow.mdc
**工作流程规则**

- 详细的工作流程步骤
- 6个阶段的完整说明
- 每个阶段的输入输出
- 阶段间的依赖关系

**适用场景**：
- 执行任务时的主要参考
- 确保不遗漏任何步骤
- 理解整体任务的推进顺序

### 3. mcp_usage.mdc
**MCP 工具使用规则**

- richtext-reader 工具的详细说明
- py 模式和 LLM 模式的使用策略
- 内容整理规范
- 错误处理方法

**适用场景**：
- 需要读取富文本文件时
- 判断使用哪种读取模式
- 整理读取到的内容
- 处理读取错误

### 4. file_organization.mdc
**文件组织规则**

- 各目录的用途和规范
- 文件命名规则
- 文件放置位置
- 引用和管理规范

**适用场景**：
- 保存文件时确定位置
- 组织项目结构
- 引用其他文件
- 检查文件完整性

### 5. experiment_completion.mdc
**实验完成规则**

- 任务分析方法
- 编程任务完成规范
- 理论任务回答要求
- 实验报告撰写指南
- 质量检查清单

**适用场景**：
- 编写代码时参考
- 撰写报告时参考
- 确保代码和报告质量
- 完成后的自我检查

## 使用建议

### 对于 AI 助手

1. **任务开始前**：阅读 `project_overview.mdc` 和 `workflow.mdc`
2. **执行任务时**：
   - 按照 `workflow.mdc` 的流程推进
   - 读取文件时参考 `mcp_usage.mdc`
   - 保存文件时参考 `file_organization.mdc`
   - 编写代码和报告时参考 `experiment_completion.mdc`
3. **任务完成后**：使用 `experiment_completion.mdc` 中的检查清单

### 对于用户

1. **了解项目**：阅读 `project_overview.mdc`
2. **理解流程**：阅读 `workflow.mdc` 了解 AI 的工作方式
3. **提供文件**：
   - 将背景资料放入 `background/` 目录
   - 将实验任务和模板放入 `experiment_report/` 目录
4. **启动 AI**：告知 AI 开始执行任务
5. **等待完成**：AI 会自动完成所有阶段
6. **补充内容**：根据 AI 的提示补充截图等内容

## 规则的更新和维护

### 何时需要更新规则

- 工作流程发生变化
- 添加新的工具或功能
- 发现规则中的错误或不足
- 用户有新的需求

### 如何更新规则

1. 确定需要修改的规则文件
2. 修改对应的 `.mdc` 文件
3. 确保修改后的规则清晰、准确
4. 如果添加新规则，在本 README 中添加说明

## 规则的优先级

当规则之间可能存在冲突时，遵循以下优先级：

1. **用户的明确指令** > 所有规则
2. **实验任务的具体要求** > 通用规则
3. **特殊情况处理** > 标准流程
4. **workflow.mdc** > 其他流程规则
5. **实验质量** > 流程速度

## 常见问题

### Q1: 规则太多，如何快速找到需要的内容？

**A**: 
- 使用文件内的标题导航
- 根据当前工作阶段选择对应的规则文件
- 使用搜索功能查找关键词

### Q2: 规则和实际任务要求冲突怎么办？

**A**: 
- 优先遵循实际任务要求
- 在报告中说明偏离标准流程的原因
- 告知用户特殊处理的情况

### Q3: 发现规则有错误或不适用怎么办？

**A**: 
- AI：在当前任务中采用合理的处理方式，任务后告知用户
- 用户：可以修改规则文件以适应特定需求
